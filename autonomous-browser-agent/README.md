# Autonomous Browser Agent - AgentFlow 🤖🔍

AgentFlow is a complete, demo-friendly browser automation environment and dashboard. It allows users to enter tasks, generates a plan using local AI (Ollama) or deterministic logic, executes steps in real-time using Playwright, and outputs logs, screenshots, and scraped text in a beautiful glassmorphic dashboard.

---

## 🛡️ Safety Boundaries & Rules

To ensure a safe environment, this project enforces the following constraints:
1. **Allowlisted Browsing**: The agent can only browse safe/demo domains:
   - `example.com`
   - `httpbin.org`
   - `wikipedia.org` / `*.wikipedia.org`
   - `developer.mozilla.org`
   - Local mock pages hosted on the server (`http://localhost:<port>/demo/*`)
2. **Demo Actions Only**: Never performs real purchases, payments, accounts, logins, CAPTCHA bypasses, or external state mutations.
3. **No Private Data**: All form submissions use mock sample data generated by the server/client.
4. **Visual "Demo Mode" Labels**: The interface clearly indicates it is operating under a safe demo context.
5. **Pre-Execution Confirmation**: The dashboard blocks execution until the user checks the confirmation disclaimer.

## 🛠️ Supported Demo Tasks

The system includes pre-defined configurations for:
1. **Search Wikipedia**: Scrapes page content for "Artificial Intelligence" and extracts the introduction.
2. **Search MDN Docs**: Visits Mozilla Developer Network to find "JavaScript fetch API" details.
3. **Contact Form Submission**: Automates filling the mock contact page (`/demo/contact.html`) with sample details.
4. **Compare Products**: Browses the mock shopping catalog (`/demo/shopping.html`) and lists details/prices for 3 products.
5. **Simple Heading Scrape**: Inspects `example.com` to extract page headers.

## ⚙️ How it Works

### 1. The Planner
When a task is submitted, the backend attempts to query a local Ollama instance (typically running `llama3.2` on port `11434`). If unreachable, it falls back to a deterministic rule-based planner that parses keywords (e.g. "wikipedia", "contact form", "shopping", etc.) and maps them to a series of structured JSON automation steps.

### 2. The Safe domains List
Every navigation step goes through a domain verification utility. If Playwright attempts to navigate to an unlisted external link (e.g. `google.com`, `amazon.com`), it blocks the request immediately with an error log to protect user safety.

### 3. The Playwright Browser Agent
Launches a Playwright browser instance, executes steps (navigation, typing, clicking, extracting), takes screenshots after important events, and streams logs/metadata back to the frontend.

## 🚀 Getting Started

### Prerequisites
- Node.js (v18+)
- (Optional) Ollama running locally

---

### Backend Setup

1. Navigate to the server folder:
   ```bash
   cd server
   ```
2. Install packages:
   ```bash
   npm install
   ```
3. Install Playwright browser binaries:
   ```bash
   npx playwright install
   ```
4. Start the backend:
   ```bash
   npm start
   ```
   *(Running on port 5000 by default)*

---

### (Optional) Ollama Setup

To run local LLM-based planning:
1. Install [Ollama](https://ollama.com).
2. Download the model:
   ```bash
   ollama pull llama3.2
   ```
3. Ensure the service is active:
   ```bash
   ollama serve
   ```

---

### Frontend Setup

1. Navigate to the client folder:
   ```bash
   cd ../client
   ```
2. Install packages:
   ```bash
   npm install
   ```
3. Start the Vite dev server:
   ```bash
   npm run dev
   ```
   *(Access at http://localhost:5173)*

---

## ☁️ Deployment Notes

- **Frontend**: The React client compiles to static HTML/JS/CSS assets and can be deployed easily on Vercel, Netlify, or GitHub Pages. Make sure to define the backend API URL environment variable `VITE_API_URL`.
- **Backend**: Because the backend runs Playwright browser instances, standard serverless platforms (like basic Vercel functions) might fail due to storage and runtime limits. Deploy the server to platforms like **Render**, **Railway**, or **render.com** inside a Docker container configured with Playwright dependencies.
